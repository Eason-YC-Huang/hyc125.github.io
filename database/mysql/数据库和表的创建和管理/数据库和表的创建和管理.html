数据库的创建和管理

在了解MySQL数据库的创建和管理之前我们需要先了解两样东西。

MySQL语句大小写规则

  1. MySQL关键字和函数名是不区分大小写的。
  2. 存储函数、过程和时间的名字都不区分大小写
  3. 列名和索引名不区分大小写
  4. 别名，默认情况下表的别名区分大小写
  5. 因为MySQL使用主机上的文件和目录来表示数据库和表，所以数据库名、表名和视图名是否区分大小写跟操作系统的文件系统有关，例如在Windows是不区分大小写，而在Linux是区分的

MySQL字符集的支持

MySQL支持多种字符集，而且允许在服务器、数据库、表、列和字符串常量等不同层次单独指定字符集。
一个给定的字符集可以有一种或多种排序规则，所谓排序规则就是该字符集是通过什么方式来进行比较的。
我们可以通过以下例子说明：
mysql> show character set like '%gbk%';
+---------+------------------------+-------------------+--------+
| Charset | Description            | Default collation | Maxlen |
+---------+------------------------+-------------------+--------+
| gbk     | GBK Simplified Chinese | gbk_chinese_ci    |      2 | 
+---------+------------------------+-------------------+--------+
1 row in set (0.00 sec)

mysql> show collation like '%gbk%';
+----------------+---------+----+---------+----------+---------+
| Collation      | Charset | Id | Default | Compiled | Sortlen |
+----------------+---------+----+---------+----------+---------+
| gbk_chinese_ci | gbk     | 28 | Yes     | Yes      |       1 | 
| gbk_bin        | gbk     | 87 |         | Yes      |       1 | 
+----------------+---------+----+---------+----------+---------+
2 rows in set (0.00 sec)
从上例中我们可以看出字符集gbk有两个排序方式（分别为gbk_chinese_ci何gbk_bin），其中默认的排序方式为gbk_chinese_ci。
排序方式的命名规则为：字符集名字_语言_后缀，其中各个典型后缀的含义如下：

1）_ci：不区分大小写的排序方式

2）_cs：区分大小写的排序方式

3）_bin：二进制排序方式，大小比较将根据字符编码，不涉及人类语言，因此_bin的排序方式不包含人类语言

因此，gbk_chinese_ci排序方式就表示：字符集为gbk，人类语言使用中文来比较大小，比较时区分大小写。

指定字符集和排序规则

在用于创建数据库和表的SQL语句里，有两个句子可用于指定数据库、表和列的字符集和排序规则。
# 指定字符集
CHARACTER SET charset
# 指定字符集的排序方式
COLLATE collation
# 例子 使用utf8字符集，区分大小写的排序方法
CREATE DATABASE learn_mysql CHARACTER SET utf8 COLLATE utf8_general_cs;

数据库的CRUD

-- 创建数据库
CREATE DATABASE [IF NOT EXISTS] learn_mysql [CHARACTER SET charset] [COLLATE collation];
-- 删除数据库
DROP DATABASE learn_mysql;
-- 查看数据库
SHOW DATABASES；
-- 使用数据库
USE db_name;
-- 修改数据库的全局属性，也就是默认字符集和排序规则
ALTER DATABASE db_name [CHARACTER SET charset] [COLLATE collation];

表的创建和管理

创建表

CREATE TABLE table_name
(
  字段名 字段类型,
  约束
)

create table student
(
  studentId int AUTO_INCREMENT PRIMARY KEY,
  name varchar(50) comment '学生姓名',
  age  int
)ENGINE=InnoDB default character set utf8mb4; -- MySQL在创建表时可以指定存储引擎, 引擎名字不区分大小写

定义约束

create table student
(
  studentId int PRIMARY KEY, -- 主键约束
  name varchar(50) not null, -- 非空约束
  age  int default 0, -- 默认值约束
  version int auto_increment, -- 自增约束
  foreign key 外键字段名称 references 目标表名(被关联的字段名称) -- 外键约束
)ENGINE=InnoDB default character set utf8mb4;

临时表

在创建表的时候加上TEMPORARY关键字，服务器将创建出一个临时表
CREATE TEMPORARY TABLE user
(
  name VARCHAR(20) ,
  birth DATE NOT NULL ,
  weight INT(11) ,
  gender ENUM('F','M')
)ENGINE=InnoDB;
-- 注意:
--   与服务器终止连接后，临时表将会消失
--   临时表只对创建该表的客户端可见
--   如果临时表与永久表的名字相同，那么在临时表消失之前，所有操作都会用在临时表身上

根据其它表或查询结果来创建表

MySQL提供了两条语句，可用于根据其它表或根据查询结果创建新表。
  • CREATE TABLE ... LIKE 根据原有表创建一个新表，该表是原有表的一个空副本，会把原有表的结构丝毫不差的复制过来。
    CREATE TABLE new_user LIKE user;
  • CREATE TABLE ... SELECT 根据查询结果创建一个新表,该表不会复制所有的列属性，如AUTO_INCREMENT列就不会被复制 注意：新的列会根据你所选择的列来命名，如果某个列是以表达式的计算结果生成的，那么该列的名字就是该表达式的问题表示，例如 为了避免这个状况，我们可以使用别名 特别是多表查询时，如果不同的表具有相同列，如果不指定列别名，那么就会报错。 前面说过这种方式创建的表不会复制所有的列属性，如果想解决这个问题，请查看cast()函数的内容。
    CREATE TABLE new_user SELECT * FROM user;
    mysql> CREATE TABLE mytbl SELECT PI()*2;
mysql> SELECT * FROM mytbl;
+----------+
| PI()*2   | # 注意这个列名
+----------+
| 6.283185 |
+----------+
    mysql> CREATE TABLE mytbl SELECT PI()*2 AS mycol;

分区表

MySQL支持表分区，让表的内容分散存储在不同的物理存储位置。
假设你想要创建一个表，用于存储日期和性别的数据，并且假设有累计多年的数据需要加载到这个表里。那么我们可以根据日期，使用范围分区的方式，按年份讲各行分配到某个给定的分区。
CREATE TABLE log_partition (
  dt   DATETIME     NOT NULL,
  info VARCHAR(100) NOT NULL,
  INDEX (dt)
)PARTITION BY RANGE (YEAR(dt)) (
  PARTITION p0 VALUES LESS THAN (2010),
  partition p1 VALUES LESS THAN (2011),
  partition p2 VALUES LESS THAN (2012),
  partition p3 VALUES LESS THAN (2013),
  partition PMAX VALUES LESS THAN MAXVALUE
);
当2014过完后
ALTER TABLE log_partition
REORGANIZE PARTITION pamx
INTO (
  partition p4 VALUES LESS THAN (2014),
  partition pamx VALUES LESS THAN MAXVALUE
);
默认情况下，MySQL会将分区存储在专属于分区表的数据库目录里。如果想存储分散到其他地方，则需要分区选项DATA_DIRECTORY和INDEX_DIRECTORY，具体使用方式上网查询。

删除表

DROP TABLE table_name;
DROP TABLE t1,t2;
DROP TABLE IF EXISTS table_name;
DROP TEMPORARY TABLE IF EXISTS table_name;

查看表

SHOW CREATE TABLE table_name；
SHOW TABLE STATUS;
DESC TABLE table_name;

修改表结构

我们可以通过ALTER TABLE语句来对表结构进行修改。
ALTER TABLE tb_name1 action [,action] ...;
其中的每个action是指对表所做的修改。

改变列的数据类型

ALTER TABLE user MODIFY password int(11);
ALTER TABLE user CHANGE password password varchar(50);
# 为什么使用CHANGE需要写两次列名？
# 因为CHANGE不仅能改变列的数据结构，还能改变列名
修改列的数据类型的一个重要原因是，提高多表连接的查询效率。索引经常用于两个相似类型之间的连接比较，如果两个类型完全相同，比较速度会更快。

更换存储引擎

ALTER TABLE user ENGINE=InnoDB;

重新命名表

ALTER TABLE user  RENAME TO old_user; 
RENAME TABLE user TO old_user,table2 TO old_table2;
