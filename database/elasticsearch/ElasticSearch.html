基本概念

  • 索引、文档、映射：索引类比数据库表、一个文档类比表里面的一条记录、映射类比表定义
  • 分片：类似于对表进行水平切分
主分片主要用以解决水平扩展的问题，通过主分片，就可以将数据分布到集群上的所有节点上，一个主分片就是一个运行的 Lucene 实例，当我们在创建 ES 索引的时候，可以指定分片数，但是主分片数在索引创建时指定，后续不允许修改，除非使用 Reindex 进行修改。 副本分片用以解决数据高可用的问题，也就是说集群中有节点出现硬件故障的时候，通过副本的方式，也可以保证数据不会产生真正的丢失，因为副本分片是主分片的拷贝，在索引中副本分片数可以动态调整，通过增加副本数，可以在一定程度上提高服务查询的性能（读取的吞吐）。
  • 节点、集群：一个运行的elasticsearch实例就是一个节点、集群由一个或多个节点组成

索引的增删改查

创建索引
# 创建一个叫school的索引
put http://localhost:9200/school
删除索引
DELETE /my-index-000001
查看所有索引
# 查看所有索引
GET _aliases?pretty=true
创建/更新文档
# _doc 说明是文档，10可以理解为文档的名称
post http://localhost:9200/school/_doc/10

基本查询
// 查询指定索引的10条记录
POST fat_ticket/_search
{
    "query": {
        "match_all": {}
    }
}

// 根据创建时间倒序找出前20条记录
POST fat_ticket/_search
{
  "sort": [{"create_time": "desc"}], 
  "from": 0,
  "size": 20,
  "query":{
    "match_all": {}
  }
}

// 根据创建时间倒序找出前20条记录的ticket_id和create_time
POST fat_ticket/_search
{
  "_source": ["ticket_id","create_time"], 
  "sort": [{"create_time": "desc"}], 
  "from": 0,
  "size": 20,
  "query":{
    "match_all": {}
  }
}
