字符集与编码规则

字符集: 字符的集合, 每个字符分配有一个唯一的 ID（学名为码位 / 码点 / Code Point）
字符编码规则：将「码位」转换为字节序列的规则（编码/解码 可以理解为 加密/解密 的过程）
例如
ASCII字符集, 字符A的码位是65
ASCII编码规则, 把字符A的码位65转换成01000001存储到内存中。

Unicode

广义的 Unicode 是一个标准，定义了一个字符集以及一系列的编码规则，即 Unicode 字符集和 UTF-8、UTF-16、UTF-32 等编码规则

UTF8

下面来看一下UTF8编码规则是怎么样的
以 知 乎 为例, 它们对应的码位如下 (码位是16进制的)：
知 77e5     二进制: 0111 0111 1110 0101 
   0020     二进制: 0000 0000 0010 0000
乎 4e4e    二进制: 0100 1110 0100 1110
UTF-8的编码规则：
  1. 单字节的字符，字节的第一位设为0，对于英语文本，UTF-8码只占用一个字节，和ASCII码完全相同；
  2. n个字节的字符(n>1)，第一个字节的前n位设为1，第n+1位设为0，后面字节的前两位都设为10，这n个字节的其余空位填充该字符unicode码，高位用0补足。
这样就形成了如下的UTF-8标记位：
0xxxxxxx
110xxxxx 10xxxxxx
1110xxxx 10xxxxxx 10xxxxxx
11110xxx 10xxxxxx 10xxxxxx 10xxxxxx
111110xx 10xxxxxx 10xxxxxx 10xxxxxx 10xxxxxx
1111110x 10xxxxxx 10xxxxxx 10xxxxxx 10xxxxxx 10xxxxxx
于是，”知 乎“就变成了：
知 77e5     二进制: 0111 0111 1110 0101   两个字节
   0020     二进制: 0000 0000 0010 0000   一个字节
乎 4e4e    二进制: 0100 1110 0100 1110   两个字节
---------------------------------------
知 11100111 10011111 10100101
   00100000
乎 11100100 10111001 10001110
