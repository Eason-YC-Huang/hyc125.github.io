本文讲解SpringMVC的初始化流程，其实主要就是IoC容器在Web容器中的启动过程。

前提知识

回调函数

什么是回调函数？回调函数就是一个通过函数指针调用的函数。如果你把函数的指针（地址）作为参数传递给另一个函数，当这个指针被用来调用其所指向的函数时，我们就说这是回调函数。
通俗来说就是方法A的参数需要接收一个方法B，然后在方法A的执行过程中，它会调用方法B完成某些操作，这时方法B就是一个回调函数。
虽然在Java中不可以传递函数，但是我们通过传递类和接口也能实现类似的操作，特别是对于函数式接口，使用上已经跟回调函数差不多了。

ServletContext与ServletContextListener

什么是ServletContext？

Web容器在启动一个Servlet应用时，会创建一个ServletContext对象，它负责存储Web应用和Web服务器的信息，以及定义了一系列方法让Servlet与Servlet容器进行交流。
ServletContext被存储在ServletConfig中，servlet容器在初始化servlet时，会把ServletConfig传递给servlet。

什么是ServletContextListener

ServletContextListener会的ServletContext的生命周期进行监听，例如到ServletContext的创建和销毁事件时，它会执行某些操作。Spring就是在监听到ServletContext的创建事件时，开始IoC容器的初始化。

WebApplicationContext

为了方便在Web环境中使用Spring IoC容器，Spring对原有IoC容器进行了扩展，提供了WebApplicationContext。
在一个Spring MVC应用中，至少存在两个WebApplicationContext，其中有且仅有一个Web应用上下文会作为根应用上下文。其它Web应用上下文都是根应用上下文的子上下文。子上下文主要存储Spring MVC使用的Bean。

ContextLoaderListener

Spring IoC容器是一个独立的模块，如果需要在Web容器中使用IoC容器，就需要先把IoC容器导入到Web容器中。简单来说就是把IoC容器初始化并存储在ServletContext中。
把IoC容器初始化并存储在ServletContext中的这个过程要怎么实现呢？
在前面我们讲了ServletContextListener，它可以在ServletContext创建时执行某些操作，我们可以利用它把IoC容器初始化并存储在ServletContext。
Spring提供了一个ContextLoaderListener，它继承ContextLoader并且实现了ServletContextListener，前者用来初始化IoC容器，后者则是在ServletContext创建时调用IoC容器初始化的方法并把IoC容器存储到ServletContext。

SPI 服务提供接口

SPI（Service Provider Interface）实际上就是“基于接口编程＋策略模式＋配置文件”组合实现的动态加载机制。框架提供一个接口，第三方实现该接口，从而对框架进行了扩展。
  1. 定义一个接口，例如Java定义了一个java.sql.Driver接口
  2. 实现该接口，例如MySQL的com.mysql.jdbc.Driver就实现了java.sql.Driver接口
  3. 在/META-INF/services/目录创建一个文本文件，文件名是接口的全限定名，文件的内容可以有多行，每行都是该接口对应的具体实现类的全限定名
  4. 使用ServiceLoader读取这些实现
ServiceLoader<Driver> drivers = ServiceLoader.load(Driver.class);
drivers.forEach(driver -> System.out.println(driver.getClass().getName()));

ServletContainerInitializer

Servlet3.1添加了ServletContainerInitializer接口，通过它我们可以对Servlet容器的启动进行一些配置，例如注册servlet或者filtes等等。
ServletContainerInitializer就是一个SPI，你能在spring-web.jar的META-INF/services目录下找到相关信息。可以看出Sprng对ServletContainerInitializer的实现是SpringServletContainerInitializer

Spring MVC 初始化流程

这里主要讲的注解配置SpringMVC应用的初始化流程。
Spring通过ServletContainerInitializer对Servlet容器的启动进行配置。在/META-INF/services/目录下，可以看到ServletContainerInitializer的实现是SpringServletContainerInitializer。

SpringServletContainerInitializer

// 该注解上声明的类/接口实现类会被作为参数传递给下面的onStartup方法
@HandlesTypes(WebApplicationInitializer.class)
public class SpringServletContainerInitializer implements ServletContainerInitializer {
   
   @Override
   public void onStartup(@Nullable Set<Class<?>> webAppInitializerClasses, 
                         ServletContext servletContext) throws ServletException {
  
       // 对容器传递过来的WebApplicationInitializer的实现进行验证，合格的放入initializers
      List<WebApplicationInitializer> initializers = new LinkedList<>();
      if (initializers.isEmpty()) {
         return;
      }
      AnnotationAwareOrderComparator.sort(initializers);
      for (WebApplicationInitializer initializer : initializers) {
         // 调用WebApplicationInitializer的onStartup方法对servletContext进行配置
         // 其实就是把servletContext给你了，你想怎么搞就怎么搞
         initializer.onStartup(servletContext);
      }
   }

}

WebApplicationInitializer

下面我们来看看WebApplicationInitializer的实现类对ServletContext做了什么,以常用的AbstractAnnotationConfigDispatcherServletInitializer为例。
/**
onStartup
定义在：SpringServletContainerInitializer
最后实现在：AbstractDispatcherServletInitializer
*/

@Override
public void onStartup(ServletContext servletContext) throws ServletException {
    // 调用AbstractContextLoaderInitializer的onStartup方法
    // 注册ContextLoaderListener，实现IoC容器的初始化并存储到ServletContext
  super.onStartup(servletContext);
    // 定义在AbstractDispatcherServletInitializer
  registerDispatcherServlet(servletContext);
}

注册ContextLoaderListener

// AbstractContextLoaderInitializer
@Override
public void onStartup(ServletContext servletContext) throws ServletException {
  registerContextLoaderListener(servletContext);
}
// AbstractContextLoaderInitializer
protected void registerContextLoaderListener(ServletContext servletContext) {
    // 创建根应用上下文
  WebApplicationContext rootAppContext = createRootApplicationContext();
    
  if (rootAppContext != null) {
        // 创建ContextLoaderListener，让它实现根应用上下文的初始化并存储到ServletContext
    ContextLoaderListener listener = new ContextLoaderListener(rootAppContext);
    listener.setContextInitializers(getRootApplicationContextInitializers());
    servletContext.addListener(listener);
  }
  else {
    // logger
  }
}

创建根应用上下文

// AbstractAnnotationConfigDispatcherServletInitializer
protected WebApplicationContext createRootApplicationContext() {
    // 获取根应用上下文的配置文件
  Class<?>[] configClasses = getRootConfigClasses();
  if (!ObjectUtils.isEmpty(configClasses)) {
        // 创建根应用上下文
    AnnotationConfigWebApplicationContext context = 
            new AnnotationConfigWebApplicationContext();      
  // 这里只是注册配置文件 根应用上下文的初始化还是在我们之前讲的ContextLoaderListener完成
    context.register(configClasses);
    return context;
  }
  else {
    return null;
  }
}
