变量声明

Rust是一门支持类型推断的静态语言，我们只需要使用let关键字即可声明变量
let name = "hyc";
let age = 24;
默认情况下，Rust声明的变量是不可变的，如果需要声明一个可变的变量，需要加上mut关键字
let name = "hyc";
name = "hdr"; // 会编译报错
let mut name = "hyc";
name = "hyc"; // 编译成功
Rust支持变量名重用，在很多编程语言中，当你在一个作用域内声明了一个变量，那么就不能再使用该变量名来声明一个变量，例如在Java，例如代码是不允许的。
public void fn(){
  int num = 1;
  int num = 2;
  String num = "2";
}
但在Rust是可以的，这种情况叫作覆盖(shadowed)，后者把面前的变量声明覆盖了
fn main() {
    let x = 5;
    let x = x + 1;
    let x = x * 2;
    println!("The value of x is: {}", x);
}
个人理解，这种特性是为了更好的支持变量默认不可变这一特性
shadowed与mut的区别，mut是允许你修改变量的值，但是值必须是统一类型的，shadowed是允许你使用同一个变量名来声明不同类型的变量
let mut spaces = "   ";
spaces = spaces.len(); // 把数字赋值给字符串类型的spaces会导致编译失败
let spaces = "   ";
let spaces = spaces.len(); // 重新定义了spaces，它是一个数字类型
在Rust里，声明常量要使用const关键字，并且要写上常量的所属类型
const MAX_VALUE: u32 = 100_000;
Constants can be declared in any scope, including the global scope, which makes them useful for values that many parts of code need to know about.
The last difference is that constants may be set only to a constant expression, not the result of a function call or any other value that could only be computed at runtime.

所有权

一般来说，内存管理主要有两种方式，垃圾回收和手动回收，Rust另辟蹊径，使用包含特定规则的所有权系统来管理内存，这套规则允许编译器在编译过程中执行检查工作，而不会产生任何的运行时开销。
简单来说就是，按照它的规则写，编译时为你生成内存管理的代码
当所有者离开自己的作用域时，它持有的值就会被释放掉
fn auto_release() {
    // 在堆上创建值hello
    let owner = String::from("hello");
    println!("{}", owner);
    // 在该方式结束后,owner离开它的作用域,值hello所占用的内存会被回收
}
Rust中的每一个值都有一个对应的变量作为它的所有者
fn ownership() {
    // 变量owner是值hello的所有者
    let owner = String::from("hello");
    println!("{}", owner);
}
在同一时间内，值有且仅有一个所有者
fn ownership() {
    let owner = String::from("hello");
    // 现在变量new_owner才是值hello的所有者
    let new_owner = owner;
    // 下面一行代码会导致编译失败,因为owner已经被废弃了
    // println!("{}", owner);
    println!("{}", new_owner);
}
引用传递
对于所有权的转移带来了一个问题
fn main() {
    let str = String::from("hello");
    // 调用trans_owner方法会把值hello的所有权转移给参数str
    // 也就是它会导致变量str被废弃
    trans_owner(str);
}

fn trans_owner(str: String) {
    println!("{}", str);
}
str被废弃了，导致接下来的代码无法使用str，怎么办呢？
你可以在trans_owner方法从新返回值hello然后重新转移给str，但是这种方式也太不方便了
我们可以通过引用传递的方式解决这个问题
fn main() {
    let str = String::from("hello");
    // &操作符用于获取一个值的引用
    // 通过引用你可以不获取值的所有权也能使用该值
    borrow(&str);
    // 因为没有发生所有权转移,str依旧有效
    println!("{}", str);
}


fn borrow(str: &String) {
    // &是不可变引用,也就是你不能对值进行修改
    // 如果想要可变引用,通过&mut操作符获取 e.g.&mut String
    println!("{}", str);
}

类型系统

Rust是基于结构体来创建类型的。
struct User {
    username: String,
    email: String,
    active: bool,
}

fn build_user(username: String, email: String) -> User {
    // 如果有同名变量,在赋值时可以简写
    return User {

面向对象

Rust支持封装，通过访问修饰符实现
Rust不支持继承（如何实现属性的复用？）
Rust支持多态（不支持继承怎么来的多态？其实多态跟继承并不是强绑定的，有相同的接口实现也是可以的嘛）
方法

// 既然是基于结构体的类型系统，不难想象，方法的定义是在结构体外部
// 结构体的方法定义在impl代码块内，每个结构体可以拥有多个impl块
impl User {
    // 实例方法 (为什么这里要把实例引用写出来,不能默认带上吗? 因为它可以分为可变引用和不可变引用0.0)
    fn say_hi(&self) {
        println!("hello I am {}", self.username);
    }

    // 注意这里用的是可变引用,只有实例的可变引用才能修改实例的指
    fn update_email(&mut self, email: String) {
        self.email = email
    }

    // 静态方法 在Rust中静态方法叫作关联函数
    fn of(username: String, email: String, active: bool) -> User {
        return User { username, email, active };
    }
}

fn main() {
    // 调用User的静态方法创建一个User实例
    let user = User::of("hyc".to_string(), "hyc@foxmail.com".to_string(), false);
    // 调用user实例的saiHi方法
    user.say_hi();
    // 因为user是不可变的,所以你没办法调用user.update_email方法,要先把user变成可变的
    let mut user = user;
    user.update_email("hdr@foxmail.com".to_string());
}
