程序的使命就是要完成一项工作(work)，一项工作由多个任务(task)组成，基于该理念Java提供了Executor框架。Executor顾名思义就是执行器，用来执行任务的，对任务提交和任务的执行进行解耦。
public interface Executor {
    // 执行任务
    void execute(Runnable command);
}

线程池

Executor的其中一种实现使用了线程池设计模式，用线程池中的线程来执行任务，实现线程复用，减少创建和销毁线程的开销。

自定义线程池

通过ThreadPoolExEcutor我们可以自定义创建线程池。
public ThreadPoolExecutor(int corePoolSize,
                          int maximumPoolSize,
                          long keepAliveTime,
                          TimeUnit unit,
                          BlockingQueue<Runnable> workQueue,
                          ThreadFactory threadFactory,
                          RejectedExecutionHandler handler) {
}
  • corePoolSize ：线程池中线程的基本数量，除非设置了allowCoreThreadTimeOut，否则，就算里面的线程是闲置的，也不会进行回收。
    // 默认为false，就算线程是闲置的也不会被回收。
// 如果设置为true，闲置的线程可以存活keepAliveTime设置的时间
private volatile boolean allowCoreThreadTimeOut;
  • maximumPoolSize ：线程池中允许存在的最大线程数。
  • keepAliveTime ： 空闲线程存活的时间
  • unit ：时间单元
  • workQueue ： 用来存放任务的队列
  • threadFactory ：用来创建线程的工厂
  • handler ： 用来处理无法执行的任务，任务队列已经满了且线程数已达到最大限制，或者任务提交到一个已经被关闭的Executor。

execute 执行流程

  1. 如果此时线程池中线程数量小于 coreSize，将会新建线程执行提交的任务。
  2. 如果此时线程池线程数量大于等于 coreSize，将会直接把任务加入到队列中。线程将会从工作队列中获取任务执行。
  3. 如果工作队列已满，将会继续新建线程。
  4. 如果工作队列已满，且当前线程数大于等于 maximumPoolSize，此时将会使用拒绝策略拒绝任务。

内置线程池

Executor框架提供了一些线程池，我们可以通过
Executors
的静态方法来创建它们。
  • newCachedThreadPool 根据实际情况创建和回收线程，不限制线程的创建数量，如果线程闲置超过60秒就会被回收
  • newFixedThreadPool 固定大小的线程池，每当有新任务提交就创建一个线程，直到线程数量达到指定的最大值，然后保持线程池的大小，如果有线程死亡了，就重新创建一个新的。
  • newSingleThreadExecutor 创建一个工作线程来执行任务，如果该线程死亡，就创建一个新的。
  • newScheduledThreadPool 一个固定大小的线程池，支持任务的延迟和定时执行。
  • newWorkStealingPool a work-stealing thread pool

线程池生命周期管理

ExecutorService是Executor的扩展，它添加了对Executor的生命周期进行管理的功能（当然还有一些方便提交任务的方法）。
线程池的生命周期分为五个阶段
  • RUNNING: 线程池创建之后的初始状态，这种状态下可以执行任务。
  • SHUTDOWN: 该状态下线程池不再接受新任务，但是会将工作队列中的任务执行完毕。
  • STOP: 该状态下线程池不再接受新任务，也不会处理工作队列中的任务，并且会中断(Interrupt)正在执行任务的线程。
  • TIDYING：该状态下所有任务都已终止，将会执行 terminated() 钩子方法。
  • TERMINATED：执行完 terminated() 钩子方法之后。
// 不再接受新任务（使用线程池拒绝策略响应）
// 等待所有任务执行完以后关闭线程池 （不会阻塞当前线程）
void shutdown();

// 不再接受新任务（使用线程池拒绝策略响应）
// 中断所有工作线程（需要工作线程配合）
// 取出工作队列中任务返回给调用者
List<Runnable> shutdownNow();

// 阻塞当前线程，等待Executor进入已终止状态
boolean awaitTermination(long timeout, TimeUnit unit) throws InterruptedException;

// 判断Executor是否处于正在关闭状态
boolean isShutdown();
    
// 判断Executor是否处于已终止状态
boolean isTerminated();
