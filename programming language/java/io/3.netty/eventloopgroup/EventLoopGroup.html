以NioEventLoopGroup为例子，EventExecutorGroup顾名思义就是管理一组EventExecutor，先看下它俩的类结构。
先来看NioEventLoopGroup，它的结构简单一点，可以看出它主要分成三部分，最顶层是Java自带的Executor框架，最底层是NioEventLoopGroup，中间是Netty对EventLoopGroup的抽象，我们重点理解中间部分做了什么。

EventExecutorGroup

它有两个核心方法，next()和shutdownGracefully()，前者用于获取EventExecutor，后者是统一关闭它所管理的所有EventExecutor。

EventLoopGroup

正式跟Channel建立联系，提供了Register方法允许你把Channel注册到EventLoop中。

AbstractEventLoopGroup

EventExecutorGroup的抽象基类，本质是做了一层代理，底层还是调用next方法获取EventExecutor，把事情委托给EventExecutor做。
MultithreadEventExecutorGroup和MultithreadEventExecutorGroup也是差不多的，都是不干活的主，就是管理和委派EventLoop做事。
